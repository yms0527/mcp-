## 🤖 什么是 MCP?
**MCP (Model Context Protocol，模型上下文协议)** 是一个开放协议，旨在实现AI模型与外部数据源、工具和服务之间的无缝集成。由Anthropic开发，MCP标准化了AI助手访问和交互各种能力的方式，使扩展AI功能变得更加简单。它的核心定位是作为大模型与底层执行环境之间的桥梁，统一规范大模型向外发送工具调用指令、传输上下文数据、接收执行结果的完整流程与数据格式，解决大模型与各类工具、服务器之间接口不统一、格式不兼容、交互逻辑混乱等问题。

MCP主要有以下特性：
- 🔗 **标准化集成**：连接AI与外部工具的通用接口
- 🛠️ **工具发现**：动态发现和使用可用功能
- 🔒 **安全优先**：内置安全考虑，确保工具执行安全
- 🌐 **跨平台**：适用于不同的AI客户端和框架

## ⚠️ 恶意的MCP服务器攻击种类分类

<div align="center">
### 🔗 共计4个大类，15个小类
</div>

本部分根据不同攻击方法将安全的MCP服务器修改为恶意MCP服务器，按照不同的系统结构将其进行系统分类，涵盖从数据传输到协议规范的各个层面。我们将攻击类型划分为4个主要类别，帮助安全研究人员和开发者理解不同攻击向量的特征和危害。

### 📡 传输层攻击

> **总结介绍**：该类别攻击主要针对数据传输过程中的安全问题，攻击者通过窃取或截获传输中的敏感凭证信息，获取未授权的访问权限。

</div>

```
🔑 凭证泄露           
```


<div align="center">

### 🧠 决策层攻击

> **总结介绍**：该类别攻击主要针对AI模型的决策过程，通过操纵模型的工具选择偏好、注入恶意提示词或干扰工具调用逻辑，使AI系统做出错误的判断或执行恶意操作。

</div>

```
🎯 工具偏好操纵攻击      
🎭 傀儡攻击       
⚡ 工具抢占攻击            
🔨 工具链开发攻击          
💬 提示词注入攻击         
```


<div align="center">

### ⚡ 执行层攻击

> **总结介绍**：该类别攻击主要针对工具执行阶段的安全漏洞，通过在工具中植入恶意代码、注入恶意命令、监听通信或逃逸安全沙箱等方式，在代码执行层面造成实际危害。

</div>

```
☠️ 工具投毒攻击            
👻 工具影子攻击            
💻 命令注入攻击          
📡 远程监听攻击            
🌐 网页投毒攻击           
🏃 沙箱逃逸攻击           
🚪 后门攻击              
```


<div align="center">

### 📜 协议层攻击

> **总结介绍**：该类别攻击主要针对MCP协议本身的规范和机制缺陷，通过滥用协议设计漏洞实现未授权访问或恶意资金转移，危害协议层面的安全性和完整性。

</div>
